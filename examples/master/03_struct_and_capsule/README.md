# 03 Struct And Capsule

第3週の実践編です。構造体の物理配置、メソッド定義、パッケージによるカプセル化を実践します。

## 目的

データをひとまとまりにし、そこに振る舞いを定義し、外から壊されにくい境界を作る方法を理解します。

## 何を確認するか

### 1. 構造体の物理配置

フィールドの順序によって構造体のバイトサイズが変わります。

CPU はデータをアライメントされた位置から読もうとするため、コンパイラはフィールド間にパディングを入れます。フィールドを大きい順に並べるとパディングが減ります。

### 2. 値レシーバとポインタレシーバの違い

- 値レシーバ: メソッド内でコピーを操作する。元の値は変わらない
- ポインタレシーバ: 実体を操作する。元の値が変わる

フィールドを変更するメソッドはポインタレシーバで定義する必要があります。

### 3. パッケージによるカプセル化

- 小文字で始まるフィールド・関数はパッケージ外から見えない
- `internal/` ディレクトリは親パッケージ以外からインポートできない
