# 02 Memory Layout

第2週の実践編です。ポインタ、値渡しと参照渡し、スライスの内部構造、文字列のバイト表現を実験します。

## 目的

Go プログラムがメモリをどう使うかを、実際に動かして確認します。

- コピーと参照のコスト差を理解する
- スライスの `append` による再確保の挙動を観測する
- 日本語文字列がバイト列として何バイト占めるかを確認する

## 何を確認するか

### 1. 値渡しと参照渡し

関数に値を渡したとき、何がコピーされるかを確認します。

- 値渡し: 変数の中身がコピーされる
- ポインタ渡し: アドレスだけがコピーされ、実体は共有される

### 2. スライスの内部構造

Go のスライスは「ポインタ・長さ・容量」の3要素で構成されます。

`append` によって容量を超えると、新しい配列が確保されて元のスライスとは別物になります。

### 3. 文字列とバイト表現

Go の `string` は UTF-8 バイト列です。

日本語1文字は3バイト占めるため、`len(s)` は文字数ではなくバイト数を返します。


## 到達基準

このサンプルを動かしたあと、次の問いに答えられれば第2週の到達点として十分です。

- 値渡しと参照渡しで、何がコピーされるのか
- `append` によるスライスの再確保はいつ起きるのか
- `len("日本語")` が `9` になる理由は何か

## 補足観点

- `unsafe.Sizeof` を使うと、構造体のバイトサイズを確認できます
- `-gcflags=-m` をつけてビルドすると、スタックとヒープのどちらに確保されるかを確認できます
