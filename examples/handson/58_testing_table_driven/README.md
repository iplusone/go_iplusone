# 58. Testing Table Driven

テーブル駆動テストの書き方を学びます。

- 入力と期待値をテーブル（構造体スライス）として定義する
- `for` ループで全ケースを実行する
- `t.Run` でサブテストに名前をつける

テスト対象を変えずに、ケースを追加するだけでカバレッジが上がります。

## 実行

```bash
go test ./examples/handson/58_testing_table_driven/...
```

## 学習ポイント

1. テーブルに `name` フィールドを入れると失敗時にどのケースかすぐわかる
2. `t.Run(tc.name, ...)` でサブテストにするとフィルタ実行できる（`-run TestAdd/正の数`）
3. 境界値（0、負の数、最大値など）をテーブルに並べやすい
