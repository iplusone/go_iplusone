# 37 Pointer Receivers: メソッドの中で元の値を変えるための書き方

このハンズオンでは、ポインタレシーバに焦点を当てます。値レシーバとの違いをはっきり見る回です。

ポインタレシーバを使うと、メソッドの中で元の値そのものを更新できます。構造体の状態を変えたいときにはこちらが自然です。

逆に値レシーバでは、コピーに対して処理するので、元の値は変わりません。この違いが曖昧なままだと、あとで「変わると思ったのに変わらない」コードに見えてしまいます。

この回は短いですが、メソッド設計のかなり重要な分岐を扱っています。状態変更を伴うならポインタレシーバ、という基準の入口として読むとよいです。
