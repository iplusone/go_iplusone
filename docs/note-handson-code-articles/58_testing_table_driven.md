# 58 Testing Table Driven: テストケースを並べて、漏れなく試す書き方

このハンズオンではテーブル駆動テストを扱います。Go のテスト文化ではかなり定番の形です。

ポイントは、入力と期待値を構造体スライスとして並べ、それを `for` で回すことです。これにより、ケースを増やしてもテストコード本体はほとんど増えません。

`t.Run` でサブテストに名前をつける形も大事です。失敗したときにどのケースが壊れたのかが分かりやすくなります。

この回は、「テストを書く」よりも、「テストケースを整理して並べる」感覚をつかむ回です。コードを守るためのかなり実用的な型です。
