# 56 I/O Reader Writer Basic: I/O をインタフェースで統一して考える

このハンズオンでは `io.Reader` と `io.Writer` を扱います。Go の I/O がきれいだと感じる人が多いのは、この統一された見え方のおかげです。

`strings.NewReader` を `Reader` として使い、`bytes.Buffer` を `Writer` として使っています。さらに `io.Copy` でその2つをつないでいます。ここで見てほしいのは、対象がファイルでなくても、同じインタフェースで流れを作れることです。

後半では `bytes.Buffer` が `Reader` にもなるので、書いたものをまた読み直せます。この柔らかさが I/O 設計の強さです。

この回は、ファイルや HTTP ボディなどがなぜ似たように扱えるのか、その入口をとてもきれいに見せています。
