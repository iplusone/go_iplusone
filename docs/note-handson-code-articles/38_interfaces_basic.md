# 38 Interfaces Basic: 型ではなく「できること」で扱う

このハンズオンから interface が出てきます。Go の interface は、クラス継承ではなく「このメソッドを持っていれば使える」という発想です。

ここで大事なのは、interface がデータの形ではなく、振る舞いの約束を表していることです。ある型が必要なメソッドを持っていれば、その interface として扱えます。

関数引数として interface を受け取る例を見ると、「呼び出し側は具体的な型を知らなくてよい」という利点が見えてきます。

この回は、Go らしい抽象化の入口です。難しく考えすぎず、「同じことができる型を同じように扱うための仕組み」と捉えると入りやすいです。
