# 14 For Init And Post: `for` の3つの場所を分けて理解する

このハンズオンは、前回の `for` をもう少し分解して見る回です。Go の `for` はシンプルですが、初期化、条件、後処理の意味を意識して読むと理解が安定します。

最初の例では、普通の `for i := 0; i < 3; i++` を動かしたあとで、コードの外から「初期化」「条件」「後処理」と言葉で説明しています。ここがこの回の中心です。

後処理を `n -= 2` に変えている例も良いポイントです。ループの更新は `i++` だけではないので、「何をどう進めるループなのか」を柔軟に表せます。

最後の、外で宣言した変数を使う例では、ループ後にも変数が残ることが確認できます。スコープがどう効くかの復習にもなっています。

この回は、「for は丸暗記するもの」ではなく「3つの役割を持つ構文」だと理解するための回です。
