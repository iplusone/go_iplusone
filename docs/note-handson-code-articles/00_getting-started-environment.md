# Goハンズオンを始める前に: 最初に用意しておきたい環境をまとめて整える

このシリーズを始める前に、先にひとつだけやっておいたほうがいいことがあります。

それは、**Go のコードを実際に動かせる環境を整えること**です。

当たり前に見えるかもしれませんが、ここを飛ばしたまま個別ハンズオンへ入ると、途中でかなり高い確率で止まります。記事を読んで内容は分かっても、手元で動かせないと理解が浅くなりやすいですし、「自分の環境だけ動かない」というところで学習の勢いが削られやすいからです。

このシリーズでは、Go の文法を1本ずつ小さく確認していく `examples/handson` と、その後に Docker・Web・DB・運用までつなぐ `examples/master` を用意しています。

ただ、最初から全部の環境を完璧にそろえる必要はありません。

まず必要なのは次の4つです。

- Go が実行できること
- エディタでコードが開けること
- ターミナルでコマンドが打てること
- Git でこのリポジトリを取得できること

さらに、このシリーズを最後まで進めるなら、途中で Docker も必要になります。  
なのでこの記事では、**Go / エディタ / Git / Docker / リポジトリ取得 / 最初の動作確認** までを一気にまとめます。

## この記事のゴール

この記事を読み終える時点で、次の状態になっていればOKです。

- `go version` が通る
- このリポジトリを手元に置けている
- `go run ./examples/handson/01_values_and_types` が実行できる
- Docker を後で使う前提が分かっている

ここまで行けば、次の記事から安心して handson を進められます。

## 1. まずGoをインストールする

最初に必要なのは Go 本体です。

公式サイト:

https://go.dev/dl/

Windows でも macOS でも、基本は公式インストーラを使うのが一番分かりやすいです。特別な理由がなければ、最新の安定版を入れて問題ありません。

インストールできたら、ターミナルで次を実行します。

```bash
go version
```

うまく入っていれば、たとえば次のような表示になります。

```bash
go version go1.xx.x windows/amd64
```

あるいは

```bash
go version go1.xx.x darwin/arm64
```

ここで大事なのは、**バージョン番号そのものより、`go version` が通ること**です。

もしコマンドが見つからないと言われたら、Go のインストールはできていても PATH が通っていない可能性があります。その場合はいったんインストーラの案内や公式ドキュメントを見直してください。

## 2. エディタを用意する

次に、コードを開くエディタを用意します。

このシリーズを進めるなら、まずは VS Code で十分です。むしろ、最初はそれが一番やりやすいと思います。

VS Code:

https://code.visualstudio.com/

入れたら、拡張機能で Go の公式拡張を追加しておくのがおすすめです。

- `Go`
  - 公開元: Go Team at Google

これを入れておくと、

- シンタックスハイライト
- フォーマット
- 補完
- エラー表示

などがかなりやりやすくなります。

ただ、最初の段階では高機能な設定を全部やる必要はありません。  
**コードが開けて、保存できて、ターミナルが使えれば十分**です。

## 3. ターミナルに慣れる

このシリーズでは、記事を読むだけでなく、ターミナルで `go run` や `go test` を打ちながら進めます。

Windows なら PowerShell、macOS ならターミナル.app か iTerm2 などで大丈夫です。

最初に最低限使えればよいのは、次の3つだけです。

```bash
pwd
ls
cd
```

Windows の PowerShell なら `pwd` や `cd` はそのまま使えます。`ls` もだいたい通ります。

ここでは、ターミナル操作を完璧に覚える必要はありません。  
**「このリポジトリの場所へ移動して、コマンドを実行する」** ことができれば十分です。

## 4. Git を入れて、このリポジトリを取得する

次に、ハンズオンの元になるリポジトリを手元へ持ってきます。

まだ Git を入れていない場合は、先にインストールしておきます。

Git:

https://git-scm.com/downloads

インストール確認:

```bash
git --version
```

そのあと、このリポジトリを clone します。

```bash
git clone <このリポジトリのURL>
cd go_iplusone
```

もし ZIP ダウンロードで持ってくる場合でも進められますが、今後の更新を取り込みやすいので、できれば `git clone` をおすすめします。

このシリーズでは、ディレクトリの中に `examples/handson` や `examples/master` がある状態になっていればOKです。

## 5. Docker は今すぐ必須ではないけれど、後で必要になる

ここは少し大事な話です。

`examples/handson` の前半は、Go 単体で動かせます。  
つまり、**最初の基礎文法編を始めるだけなら Docker はまだ必須ではありません。**

ただし、このシリーズ全体としては Docker をかなり重視しています。

理由は、後半で次のようなことをやるからです。

- Go アプリをコンテナで動かす
- DB を別コンテナで立てる
- フロント、アプリ、DB を分ける
- Linux 基準で環境差を減らす

なので、今すぐ使わないとしても、いずれ Docker は必要になります。

先に入れておきたい場合は、ここで入れてしまって大丈夫です。

Docker Desktop:

https://www.docker.com/products/docker-desktop/

インストールできたら確認:

```bash
docker --version
docker compose version
```

もし今は Docker を後回しにしたいなら、それでも構いません。  
その場合は、まず `examples/handson` を進めて、`examples/master` に入る前に Docker を整える流れでも十分です。

## 6. 最初の動作確認をする

ここまでできたら、いよいよ最初のハンズオンを1本動かします。

リポジトリのルートで、次を実行します。

```bash
go run ./examples/handson/01_values_and_types
```

うまくいけば、値と型の一覧がずらっと表示されます。

たとえば、

- `Go Hands-on (string)`
- `20 (int)`
- `3.14 (float64)`
- `true (bool)`

のような表示が見えるはずです。

ここで大事なのは、**結果を全部理解することではなく、まず手元でGoコードが動いたことを確認すること**です。

この最初の一歩が通ると、以後の handson はかなり進めやすくなります。

## 7. もしここで詰まったら、見るべき場所

最初の段階で止まりやすいのは、だいたい次のどれかです。

### `go version` が通らない

Go のインストールか PATH 設定を見直します。

### `git clone` ができない

Git のインストール状況、またはリポジトリURLを確認します。

### `go run` でエラーになる

今いるディレクトリがリポジトリルートか確認します。  
`pwd` で場所を見て、`ls` で `examples` ディレクトリがあるか確かめると切り分けやすいです。

### エディタでは見えるのに、ターミナルで動かない

ターミナルが別の場所を向いていることが多いです。  
`cd` でリポジトリルートへ移動してから、もう一度試します。

## 実行結果の見どころ

- `go version` が通ること自体が最初のチェックポイント
- `go run ./examples/handson/01_values_and_types` が通れば、以後の handson に入る準備ができていること
- Docker はこの時点では未使用でも、後で必要になる位置づけだと分かること

## 試してほしい確認

- `go version` を実行して、Go のインストール確認をする
- `git --version` を実行して、Git が使えるか確認する
- `go run ./examples/handson/01_values_and_types` を実行して、最初のハンズオンを動かす
- 余裕があれば `docker --version` と `docker compose version` も確認する

## まとめ

Go のハンズオンを始める前に必要なのは、ものすごく大がかりなセットアップではありません。

まずは、

- Go が入っている
- エディタがある
- ターミナルが使える
- リポジトリが手元にある
- 最初の `go run` が通る

ここまでできれば十分です。

Docker はこのあと必要になりますが、最初の一歩としては、まず Go コードを1本動かしてみるところまで行ければOKです。

次からは、実際に `examples/handson` を1本ずつ追いながら、Go の基礎を積み上げていきます。
