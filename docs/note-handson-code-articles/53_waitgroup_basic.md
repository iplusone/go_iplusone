# 53 WaitGroup Basic: goroutine の「終わり」をちゃんと待つ

このハンズオンでは `sync.WaitGroup` をとても小さな形で確認します。goroutine を起動するだけでは十分ではなく、完了を待つ仕組みが必要だとよく分かる回です。

`wg.Add(1)` で待つ件数を増やし、goroutine 側で `defer wg.Done()`、最後に `wg.Wait()`。この3点セットが基本形です。

Go では、main 関数が終わればプログラム全体が終わります。だから、起動した goroutine が終わる保証はありません。WaitGroup はそこを人間が明示するための道具です。

この回は短いですが、並行処理を安定して書くための必須パターンをかなり分かりやすく示しています。
