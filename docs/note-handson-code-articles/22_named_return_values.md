# 22 Named Return Values: 戻り値に名前がつくと、関数の意図が少し見えやすくなる

このハンズオンでは、名前付き戻り値を扱います。Go では必須ではありませんが、コードを読むと時々出てくるので、一度見ておく価値があります。

名前付き戻り値は、関数宣言の時点で「何を返すのか」に名前を付ける書き方です。関数の中でその変数へ値を入れて、最後に `return` だけ書くこともできます。

この回で大事なのは、「便利だから全部これにする」ではなく、「戻り値の意味をはっきり見せたい場面ではこういう書き方もある」と理解することです。

Go では書きすぎるとかえって読みにくくなることもあるので、名前付き戻り値は節度を持って使うのが自然です。このハンズオンは、その存在と基本の動き方を知るための回だと思うとちょうどよいです。
