# 55 Race Detector And Atomic: 競合は見えにくいから、道具で見に行く

このハンズオンでは、`sync/atomic` とレース検出器の話を扱います。見えにくいデータ競合を、どう意識するかの入口です。

サンプルコード自体は `atomic.AddInt64` と `Mutex` の安全なカウンタを見せています。どちらも最終結果が壊れない書き方です。

README で特に大事なのは、`count++` が「読む・足す・書く」に分かれるため競合しうる、という説明です。1行に見えても不可分ではない。この感覚が並行処理では本当に重要です。

`go run -race` という道具があることも、ぜひ覚えておきたいです。競合は目で見るだけでは分かりにくいので、検出器の存在はかなり心強いです。
