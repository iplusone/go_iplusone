# 40 Interface Collections: 違う型を同じ並びで処理できる気持ちよさ

このハンズオンでは、同じ interface を実装した複数の構造体を1つのコレクションへ入れます。Go の interface の便利さが、かなり実感しやすい回です。

型の中身や表示方法が違っても、共通メソッドだけで処理を進められるので、「違うものを同じ流れで扱う」感覚がつかめます。

ここで大事なのは、配列やスライスの中に具体型ではなく interface を入れている点です。これによって、外側の処理がかなりすっきりします。

この回は、抽象化が単なる概念ではなく、「ループの書き方そのものをシンプルにする」ことがよく分かる回です。
