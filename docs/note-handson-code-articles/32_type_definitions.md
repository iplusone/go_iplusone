# 32 Type Definitions: 同じ元型でも「別の意味の型」を作れる

このハンズオンでは `type` を使った独自型定義を扱います。Go では `int` や `string` をそのまま使うだけでなく、「意味の違う値」として別型にできます。

ここで大事なのは、元の型が同じでも、新しく定義した型は別物として扱われることです。単なる別名ではなく、意味を分けるための仕組みとして使えます。

構造体のフィールドで独自型を使う例を見ると、データに少しずつ意図が乗ってくるのが分かります。

この回は派手ではありませんが、「型で意味を表す」という Go の設計感覚を学ぶ入口としてかなり大事です。
