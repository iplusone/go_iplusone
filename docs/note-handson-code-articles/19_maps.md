# 19 Maps: キーで値を引けるようになると、データの持ち方が変わる

このハンズオンでは `map` を扱います。スライスが順番付きの集まりだとすると、map は「名前やキーで値を引くための箱」です。

最初のリテラルで `scores` を作っている部分は、Go で map を初期化する基本形です。ここでは文字列キーに整数値を持たせています。

次に追加と更新があります。`scores["Java"] = 82` のように、新しいキーを入れるのも、既存キーの値を書き換えるのも同じ書き方でできます。ここは配列やスライスとの違いが大きいところです。

参照の部分では、存在するキーと存在しないキーの扱いを意識して読むのが大事です。Go では `value, ok := m[key]` で「値」と「存在したか」を一緒に取る形がよく出ます。

最後の `range` は、map も繰り返せることを見せています。ただし順番は保証されないので、そこも含めて「キーによる参照」が主役だと理解すると分かりやすいです。
