# 39 Interfaces With Structs: 異なる構造体を同じ入口で扱う

このハンズオンでは、構造体にメソッドを生やして interface として扱う流れを見ます。前回より少し実践寄りです。

ポイントは、異なる構造体でも、同じ interface を満たしていれば同じ形で扱えることです。呼び出し側から見えるのは「その型が何者か」ではなく、「何ができるか」になります。

また、interface 型では定義された機能だけが見えるという点も重要です。具体型がもっと多くの機能を持っていても、interface 経由では約束した部分だけを扱います。

この回は、設計を柔らかくする第一歩としてかなり大事です。型の違いを超えて処理の流れをそろえられる感覚がつかめます。
