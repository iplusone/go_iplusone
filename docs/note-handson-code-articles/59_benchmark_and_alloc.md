# 59 Benchmark And Alloc: 速い遅いを感覚ではなく数字で見る

このハンズオンではベンチマークとアロケーション計測を扱います。実装の良し悪しを、雰囲気ではなく数字で判断する入口です。

`BenchmarkXxx` 関数で `b.N` 回ループし、`go test -bench=. -benchmem` で計測します。ここで出る `ns/op` `B/op` `allocs/op` は、それぞれ時間、メモリ量、確保回数の目安です。

Go ではヒープ確保が多いと GC 負荷が上がりやすいので、単に速さだけでなく `allocs/op` も重要です。

この回は、最適化を始める前に「まず測る」という文化に入る回です。性能の話をちゃんと現実に戻してくれます。
