# 43 Type Assertions: 空の interface から元の型を取り戻す

このハンズオンでは、型アサーションを扱います。空の interface に何でも入れられるようになると、その次に必要になるのが「どうやって元の型として使うか」です。

`value.(int)` や `value.(string)` の形で中身の型を指定できますが、このハンズオンでは安全な `v, ok := ...` も見せています。ここがとても大事です。

空の interface は柔軟ですが、そのままでは具体的な操作ができません。型アサーションで元の型へ戻して初めて、整数として計算したり、文字列として処理したりできます。

この回は、空の interface を使ったあとに必ず出てくる「取り出し」の基本を確認する回です。
