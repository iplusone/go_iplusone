# 20 Functions Basic: ここから「処理を名前で呼ぶ」世界に入る

このハンズオンでは、Go の関数定義の基本を見ます。ここからコードは長くなるのではなく、むしろ整理され始めます。

関数は「処理に名前をつけて再利用する仕組み」です。引数なし、引数あり、戻り値ありというパターンを順番に見ることで、Go の関数の見た目に慣れていきます。

この回で特に大事なのは、関数シグネチャです。関数名だけでなく、「何を受け取り、何を返すか」がコードの約束になります。

Go では型がかなりはっきりしているので、関数も `func name(arg type) returnType` の形で素直に読めます。この読み方が身につくと、他人のコードもかなり追いやすくなります。

この回は、関数を書く技術というより、処理を分けて考える入口の回です。
