# 17 Arrays And Slices: 配列とスライスの違いを最初に体でつかむ

このハンズオンは、Go 学習のひとつの山場です。配列とスライスは見た目が似ていますが、役割も使い勝手もかなり違います。

最初の `numbers := [3]int{...}` は配列です。長さが型の一部なので、固定長の値として扱われます。ここでは `%T` を出しているので、`[3]int` という型そのものを確認できます。

次に `range` で配列を回すことで、添字と値を取り出す基本を見ます。これは後でスライスにもそのまま使えます。

後半の `scores := []int{...}` がスライスです。ここで初めて `len` と `cap` が出てきます。長さだけでなく容量という考え方があるのが、スライスの特徴です。

`append` で要素を増やしたあとに `len` と `cap` を見直しているのも重要です。スライスは固定長ではなく、伸びる値だと実感できます。

最後の部分スライス `scores[1:3]` は、スライスが一部を切り出して扱えることを見せています。この回は、「Go で可変長の集まりを扱うならスライスが中心」と理解する入口です。
