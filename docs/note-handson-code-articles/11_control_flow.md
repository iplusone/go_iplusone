# 11 Control Flow: 条件分岐が入るとコードに判断が生まれる

このハンズオンでは、`if` `else` `switch` をまとめて確認します。ここからコードが「ただ上から順に動く」だけでなく、条件に応じて流れを変えるようになります。

最初の `if / else` はとても基本的ですが、条件式が `bool` であること、条件によって分岐が変わることを素直に見せています。

比較演算子を並べている部分は、Go で条件を書くための材料をまとめて確認するパートです。後でループやエラーハンドリングにもそのまま出てきます。

ショートステートメント付き `if` は Go らしい書き方です。条件判定のためだけに使う変数を、その場で閉じ込められるので読みやすくなります。

最後の `switch` は、単なる if の代わりではなく、「同じ値に対して複数の候補を分ける」書き方として見ると理解しやすいです。条件なし `switch` も、段階的な判定を書くときにかなり便利です。
