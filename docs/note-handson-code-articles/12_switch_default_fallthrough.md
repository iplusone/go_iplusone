# 12 Switch Default Fallthrough: `switch` がどこで止まるかを理解する

このハンズオンでは、`switch` の中でも少し混乱しやすい `default` と `fallthrough` を扱います。

最初の `default` はシンプルです。どの `case` にも一致しなかったときの逃げ道として使います。ここは他言語の `default` と似ています。

次に大事なのが、Go の `switch` は一致した `case` を実行したら、基本的にはそこで終わるという点です。C系言語に慣れていると、次へ落ちる印象を持ちがちですが、Go は逆です。

だからこそ `fallthrough` が特別な意味を持ちます。これを書くと、次の `case` の条件を再評価せず、そのまま次の処理へ進みます。ここがこのハンズオンの一番大事なポイントです。

`fallthrough` は便利というより、挙動をはっきり理解してから使うべき機能です。この回では「通常は落ちない、`fallthrough` を書いたときだけ次へ進む」と覚えておくと十分です。
