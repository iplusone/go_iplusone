# GoSpec学習ガイド

このドキュメントは、Go 言語仕様書の日本語訳 `gospec-ja` を入口として、初心者が Go の言語仕様を効率よく学ぶための全体ガイドです。

前提として、言語仕様書は「入門文法を最初から教える教材」ではなく、「Go コンパイラがコードをどう解釈すべきか」を定義する設計図です。そのため、基本文法は入門教材で押さえたうえで、仕様書は `なぜこのコードはこう動くのか` を確認する辞書として使うのが最も効果的です。

この版では、単なる読書順メモではなく、`教材としてどう進めるか` を意識して、章立て、学習目標、演習の軸まで含めて整理しています。

参照先:

- 日本語訳: <https://hiwane.github.io/gospec-ja/>
- 公式最新版: <https://go.dev/ref/spec>

注意:

- `gospec-ja` は便利な日本語訳ですが、掲載されているのは 2021 年 2 月時点ベースの内容です。
- Generics など、その後の Go の仕様変更を確認したい場合は、必ず公式最新版も参照してください。

## このガイドの使い方

- まず全体構成を把握する
- 次に初心者向けの推奨順で読む
- 各章を読んだら、必ず最小コードで挙動を試す
- わからない用語は「重要用語」節に戻って確認する
- 最新仕様が気になる箇所は公式英語版で裏を取る

## 対象読者

- Go の基本文法は一通り触ったが、挙動の理由をうまく説明できない人
- コンパイルエラーや型エラーを「雰囲気」ではなく「仕様」で理解したい人
- GoSpec を読んでみたいが、どこから入ればよいかわからない人

## 学習の前提

- `if` `for` `func` `struct` `slice` `map` の基本文法は、入門教材で軽く触れておく
- 仕様書は最初から全文読破しようとしない
- 1 つ読んだら、必ず最小コードで挙動を確認する

## 教材としてのゴール

このガイドを 1 周した時点で、次の問いに自力で答えられる状態を目指します。

- なぜ `int` と `MyInt` は別の型として扱われるのか
- なぜ `[]int` を `type MySlice []int` に代入できることがあるのか
- なぜ map 要素には `&` を取れないのか
- なぜ可変長引数に何も渡さないと `nil` になるのか
- なぜ `init()` は `main()` より先に実行されるのか
- なぜ `select` で複数の channel 操作を扱えるのか

## 先に押さえるべき重要用語

GoSpec は用語がわかると急に読みやすくなります。まず次の用語を優先して理解してください。

### `underlying type`

型の本体です。`type MyInt int` なら `MyInt` の基底型は `int` です。

### `named type`

名前を持つ型です。仕様上、名前付き型は他の型と厳密に区別されます。

### `assignability`

ある値をある型の変数に代入できるかを定める規則です。

### `representability`

ある定数値が、その型の値として表現可能かどうかを表します。

### `addressable`

`&` を取れるかどうかを表す性質です。変数やスライス要素は典型例ですが、マップ要素は違います。

### `untyped constant`

文脈に応じて型が決まる定数です。`const a = 2` の `2` は型なし整数定数です。

## 実際の進め方

1 つの節を読んだら、必ず次の 5 ステップで確認します。

1. 該当節を 15〜30 分読む
2. 重要用語を 1〜2 個だけメモする
3. 5〜10 行の最小コードを 2〜3 本書く
4. `OK ケース` と `NG ケース` を 1 本ずつ試す
5. `なぜそうなるか` を 1 文で言い換える

このやり方だと、仕様を「読むだけ」で終わらせず、デバッグ力に変換できます。

## 各章で共通してやること

教材として進めるときは、毎章この 4 点を必ずセットにします。

1. 仕様書の該当節を読む
2. 重要用語を 3 個までに絞って定義する
3. 最小コードを 2〜3 本書く
4. `なぜそうなったか` を 1 文で説明する

この 4 点を繰り返すだけで、仕様書が `読むだけの資料` から `デバッグの基準` に変わります。

## 章ごとの読み方のコツ

### `表記`

EBNF を完璧に覚える必要はありません。最初は次だけで十分です。

- `|`
  「または」
- `[]`
  省略可能
- `{}`
  繰り返し

### `型と値の性質`

ここは最重要です。最初から全部を暗記するよりも、

- 型が同じとはどういうことか
- 基底型とは何か
- 代入可能性がなぜ必要か

を自分の言葉で説明できるようになることを優先してください。

### `式`

Go 日常コードで最も効きます。特に次は重点的に読んでください。

- `呼び出し`
- `引数に ... パラメーターを渡す`
- `アドレス演算子`
- `変換`
- `評価順序`

### `文`

制御構文は知っていても、仕様の意味は意外と曖昧になりがちです。特に次が実務で重要です。

- `go 文`
- `select 文`
- `defer 文`

### `プログラムの初期化と実行`

ここを読むと、`なぜ init の順番がこうなるのか` `なぜ zero value がこう振る舞うのか` がはっきりします。

## よくあるつまずき

### 仕様書を最初から最後まで通読しようとする

最初は非効率です。必要な章から読む方が定着します。

### 教材の説明語と仕様の用語がずれる

とくに `defined type` と `named type` の混同には注意してください。教材では簡略化されがちですが、仕様を正確に読むなら `named type` を意識する方が安全です。

### コードを試さずに読んで終わる

仕様学習は `読む` だけでなく `壊して確かめる` ところまでがセットです。

### 日本語訳だけを正本だと思う

`gospec-ja` は優れた入口ですが、最終的な正本は公式仕様です。新機能や言い回しの厳密確認は英語版を基準にしてください。

## 最終到達目標

この学習を終えた時点で、次のことを自分の言葉で説明できれば十分です。

- なぜ `int` と `MyInt` は別物なのか
- なぜ `[]int` は `MySlice` に代入できることがあるのか
- なぜ map 要素には `&` を取れないのか
- なぜ可変長引数に何も渡さないと `nil` になるのか
- なぜ `init()` は `main()` より先に動くのか
- なぜ `select` は複数の channel 操作を扱えるのか
